      Forms : autocomplete input

      Working Demo:

      You selected {{autoCompleteModel1.title}} which has a value of {{autoCompleteModel1.originalObject[meta.autoCompleteRemoteCall.valuefield]}}.

      Notes:

      • Based on the angucomplete-alt directive
      • Suppport both local data and remote api calls to get the data
      Meta properties:
      • searchfields : The fields from your local data to search on (comma separate them). Each field can contain dots for accessing nested attribute
      • titlefield: The name of the field in the JSON objects returned back that should be used for displaying the title in the autocomplete list. Note, if you want to combine fields together, you can comma separate them here (e.g. for a first and last name combined). If you want to access nested field, use dot to connect attributes (e.g. name.first).
      • valuefield: The name of the field in the JSON objects returned back that should hold the item value
      • minlength: The minimum length of string required before searching. If set to 0, it shows all items. It works both local and remote but is intended to use with local data. If used with remote API, it needs to return all items when query parameter is empty string.
      • remoteItemsWrapper: The name of the field in the JSON object returned back that holds the Array of objects to be used for the autocomplete list
      • items: The hardcoded local data items in json format
      • dataSource: name/value pointing to Lexus build-in dataSource items resolution
      • dropdownStyle: Extra inline styles applied to the suggestion dropdown as an addition to the default styles within angucomplete_dropdown class. This serves as a workaround for the allignment issue on the job location typeahead control
      Model attribute:
      • The actual object that was selected is accessible in model.originalObject property
      Other attributes
      • options: Takes an object. Right now only 'class' property is being supported.
        Usage: option="{class:'mb0'}". This will append classes to the wrapper div '.form-group'
      • input-class: The names of the classes applied to the input control as a string "form-control input-sm"
      • remote-api-handler: This gives a way to fully delegate handling of remote search API. This function takes user input string and timeout promise, and it needs to return a promise. For example, if your search API is based on POST, you can use this function to create your own http handler
      • remote-response-formatter: A function on the scope that will modify raw response from remote API before it is rendered in the drop-down. Useful for adding data that may not be available from the API. The specified function must return the object in the format that angucomplete understands.
      • search-string: Property getting/setting the search input control value. It's in sync with the input control value as opposed to the model property that is only set when one of the sugguested item is selected.

      HTML:

      <ocs-autocomplete meta="meta.autoCompleteRemoteCall" 
                    input-class="form-control"
                    model="">
                    </ocs-autocomplete>

      Meta:

      autoCompleteRemoteCall:{{meta.autoCompleteRemoteCall | json}}

      Modal Popup

      Working Demo:

      Notes:

      • This creates a modal for usage by the screen. The HTML for the modal is defined in the html of the page. It also allows the ok and cancel buttons to call back to function in the active controller.
      • A couple of items to keep in mind:
        • It needs a button that on the click activates the shown value on the modal directive.
        • The name attirbute needs to mach the id of the script tag that holds the HTML of the modal.
        • In the modal the Ok and Cancel button need to call the ok() and cancel() functions. These in turn then call the ok-modal and cancel-modal functions defined on the directive.
        • To get information from the modal into the page controller you will need a couple of things:
          • Define the elements in the page controler
          • Bind that element in the html of the modal
          • Then to access it you will need to use the $rootScope.modalScope object

      HTML:

      <button type="button" class="btn btn-info" ng-click="showDocumentationModal=true">Modal Button</button>  
                        <modal shown="showDocumentationModal" name="documentationModal.html"></modal>
                        <script type="text/ng-template" id="documentationModal.html">
                        <div class="modal-header">
                        <h4 class="modal-title"><strong>Header HTML</strong></h4>
                                                </div>
                        <div class="modal-body">
                                        <ocs-input meta="meta.modalName" model="modalTest"></ocs-input>
                                                </div>
                        <div class="modal-footer">
                        <button class="btn btn-warning" type="button" ng-click="ok()">OK Button</button>
                        <button class="btn btn-warning" type="button" ng-click="cancel()">Cancel Button</button>
                        </div>
                    </script>
